&lt;h2&gt;Let Node.js scrape the page and get the title, description, keywords and image tags for you&lt;/h2&gt;
&lt;p&gt;I recently built a web-based bookmarking application for myself using React. I was happy with the results and use the app every day. However, I found myself having to go back and forth between the app&amp;#8217;s tab and the tab of the page I am bookmarking. First the URL, then the title, then grab an image from the page, and then manually enter keywords that make sense to me. Too much work. So I started to think that this was a perfect opportunity for some web page scraping.&lt;/p&gt;
&lt;p&gt;&lt;strong&gt;On a high-level the game plan was:&lt;/strong&gt;&lt;/p&gt;
&lt;ul&gt;
&lt;li&gt;The user makes a &lt;strong&gt;POST&lt;/strong&gt; request to the route:&nbsp;&lt;strong&gt;/scrape&lt;/strong&gt;, passing a &lt;strong&gt;URL&lt;/strong&gt; in the request body&lt;/li&gt;
&lt;li&gt;Secondly, we make a 2nd &lt;strong&gt;HTTP&lt;/strong&gt; request behind the scenes, to the URL provided by the user&lt;/li&gt;
&lt;li&gt;Next, we take the response of that &nbsp;2nd &lt;strong&gt;HTTP&lt;/strong&gt; request and parse it out&lt;/li&gt;
&lt;li&gt;Finally, we take various values scraped from the HTML and return it to the user as a &lt;strong&gt;JSON&lt;/strong&gt; response&lt;/li&gt;
&lt;/ul&gt;
&lt;h2&gt;Example Web Page&lt;/h2&gt;
&lt;p&gt;So, for demonstration purposes, I decided to create a simple web page that makes it easy to construct the initial HTTP POST request. Consequently, if you clone the Github repo below and follow the instructions, you can run the web page locally, enter a URL and then see the metadata scraped from the 2nd &lt;strong&gt;HTTP&lt;/strong&gt; request presented in the web page.&lt;/p&gt;
&lt;p&gt;&lt;a href="https://sub1.kevinchisholm.com/blog/images/node-cheerio-web-scrape-example.png"&gt;&lt;img class="aligncenter size-medium" style="padding: 5px;background-color: #ededed" src="https://sub1.kevinchisholm.com/blog/images/node-cheerio-web-scrape-example.png" width="500" height="572" /&gt;&lt;/a&gt;&lt;/p&gt;

&lt;p&gt;In Example # 2,&nbsp;we have the rest of the code that goes in the handler for the behind-the-scenes request. First of all, the&nbsp;&lt;strong&gt;resObj&lt;/strong&gt; variable represents the object that will be returned to the user as JSON. Furthermore, the &lt;strong&gt;$&lt;/strong&gt; variable is an important one. It represents a jQuery-like function that allows us to query the DOM of the HTML returned by the 2nd HTML request.&lt;/p&gt;
&lt;h2&gt;Creating the metadata for the JSON response&lt;/h2&gt;
&lt;p&gt;Following the&nbsp;&lt;strong&gt;$&lt;/strong&gt; variable, we create the variables:&nbsp;&lt;strong&gt;$title&lt;/strong&gt;, &lt;strong&gt;$desc&lt;/strong&gt;, &lt;strong&gt;$kwd&lt;/strong&gt;, &lt;strong&gt;$ogTitle&lt;/strong&gt;, &lt;strong&gt;$ogImage&lt;/strong&gt;, &lt;strong&gt;&nbsp;$ogkeywords&lt;/strong&gt; and &lt;strong&gt;$images&lt;/strong&gt;. The first six variables represent meta data scraped from the &lt;strong&gt;HEAD&lt;/strong&gt; section of the HTML. On the other hand, the&nbsp;&lt;strong&gt;$images&nbsp;&lt;/strong&gt;variable differs a bit in that the values in that HTML collection are scraped from the &lt;strong&gt;BODY&lt;/strong&gt; tag of the page.&lt;/p&gt;
&lt;p&gt;&lt;span style="color: #ff0000"&gt;&lt;strong&gt;View the full code for app.js here:&nbsp;&lt;a style="color: #ff0000" href="https://github.com/kevinchisholm/node-cheerio-web-page-scraper/blob/master/app.js" target="_blank" rel="noopener noreferrer"&gt;github.com/kevinchisholm/node-cheerio-web-page-scraper/blob/master/app.js&lt;/a&gt;&lt;/strong&gt;&lt;/span&gt;&lt;/p&gt;
&lt;p&gt;Over the course of the next few dozen lines of code, we just check to see if each variable has a value. If it does, we therefore add it to the&nbsp;&lt;strong&gt;resObj&lt;/strong&gt;&nbsp;object. In other words, we want to avoid any errors as we construct our JSON response. Similarly, for the&nbsp;&lt;strong&gt;$images&nbsp;&lt;/strong&gt;variable, we first make sure that the collection has length. Secondly, we use a for-loop to gather up all of the image href values and add them to the images property of the&nbsp;&lt;strong&gt;resObj&lt;/strong&gt;&nbsp;object, which is an array.&lt;/p&gt;
&lt;h2&gt;Summary&lt;/h2&gt;
&lt;p&gt;For this article, I&nbsp; focused on how the &lt;strong&gt;/scrape&lt;/strong&gt; route handler is set up to handle an HTTP &lt;strong&gt;POST&lt;/strong&gt; request and then how the 2nd HTTP request is made and subsequently handled.&lt;/p&gt;
&lt;p&gt;To be sure, there is a fair amount of code that I left out for the sake of brevity. Mainly, I did not discuss &lt;strong&gt;package.json&lt;/strong&gt;, the variable declarations at the top of &lt;strong&gt;app.js,&lt;/strong&gt; or the contents of the&nbsp;&lt;strong&gt;www&lt;/strong&gt; folder because that might have made the article quite long. If you clone this repo, follow the instructions and then run the app, it should be very easy to follow the code and understand what is happening .&lt;/p&gt;
